# 2026-06-05 — Импортёр Block-2 стал идемпотентным (фикс UNIQUE)

## Симптом
На реальных ЕГРН-данных (126 PDF + 17 XML) `import_block2` падал:
`UNIQUE constraint failed: entities.graph_node_id` (`room:61:44:0040713:446`).

## Причина
Демо и реальные данные заливались в **одну** `ekcelo.db`; кад. номер `…:446` совпал
(демо «Помещение 8» vs реал «Квартира») → одинаковый `graph_node_id`. Импортёр
создавал узлы/рёбра без проверки существования (кэш был только в рамках сессии).

## Фикс — идемпотентность
- `get_entity` — get-or-create по `graph_node_id` (кэш сессии → БД → создать).
- `add_relation` — дедуп по (from, to, relation_type); существующее ребро переиспользуется.
- `LegalRelation` 1:1 — добавляется только если ещё нет (`session.get`).
- Субъекты — get-or-create по ИНН; правообладатели без ИНН — стабильный
  `graph_node_id = subj:holder:<id>` (не случайный UUID).
- Геометрия — дедуп по `cad_number` / `entity_id`.

## Проверено (SQLite)
- 2-й и 3-й импорт демо → `entities/relations/subjects/geometries = 0`, без падений;
  граф стабилен 6/4. Сценарий «демо + пересекающийся реал :446» — не падает.
- Новый замок `test_import_block2_idempotent`. `pytest` — **3 passed**.

## Инструкция
`db-testing-economist.md`: трек B — сброс `ekcelo.db` перед реальными данными +
пометка, что импортёр идемпотентен.

## Примечание
Счётчик `objects` в выводе — «обработано» (merge), не «добавлено»; реальную дельту
показывают `entities/relations`. На граф не влияет.
