## Решения

- **enforce_rbac default False.** Навешивание require_action на /objects и др.
  сломало бы ~30 endpoint-тестов без auth. Флаг делает enforcement opt-in:
  существующие тесты/деплои не затронуты, прод включает явно.
- **/catalog НЕ защищён require_action.** Это листинг (не per-resource);
  require_action требует path-параметр resource_id, которого у /catalog нет.
  Фильтрация каталога по грантам — отдельная задача (M5, если понадобится).
- **`_verify` теперь возвращает username (был bool).** Нужно чтобы
  middleware знал кого класть в Subject. Все вызовы внутри auth.py, внешних
  потребителей нет — surgical change. Auth-тесты зелёные.
- **Basic Auth subject injection в request.state.** Унифицирует источник
  Principal: и OIDC (cycle 14 OAuthMiddleware), и Basic (M4) кладут
  `request.state.subject`. `get_principal` (M3) работает с обоими без
  спец-кода.
- **roles-карта с `|` разделителем.** `root:superadmin|assessor` — несколько
  ролей одного пользователя. Запятая разделяет пользователей, пайп — роли.
- **Порядок: auth middleware → require_action dependency.** Без credentials
  → 401 (auth), с credentials но без гранта → 403 (RBAC). Семантически
  корректно (сначала «кто ты», потом «что можно»).

## C6 ROLES_SPEC — финальное покрытие

| C6 требование | Реализация |
|---|---|
| superadmin всё | `can()` bypass (M1) + enforce (M4) |
| assessor scoped + delegate | `can()` + `delegate()` (M1) + `POST /grants` (M3) |
| client read-only + share | client hard-deny + `share()` (M1) + `POST /grants` view (M3) |
| гранты в БД (вне ЕГРН) | отдельная access.sqlite (M2) |
| enforcement в API | `require_action` на роутах (M4) |
| revoke | `DELETE /grants/{id}` (M3) |
