# 2026-06-05 — Парсер выписок ФНС ЕГРЮЛ/ЕГРИП (XML) + версионный реестр XSD

## Суть
Заказчик приложил XSD ФНС: ЕГРЮЛ v4.08 (`VO_RUGF_2_311_26_04_08_01`) и
ЕГРИП v4.07 (`VO_RIGF_2_311_27_04_07_01`). Добавлен парсер XML-выписок ФНС
(реестр юрлиц/ИП — НЕ Росреестр) с автоопределением формата и версионированием.

## Сделано
- **XSD собраны в отдельные папки по реестру** (по конвенции `xsd/upd`):
  `parser/schema/xsd/egrul/` и `parser/schema/xsd/egrip/` — оригиналы в
  windows-1251 + `NOTES.md` (источник, детект версии, правила обновления редакций).
  Новые редакции кладутся рядом, берётся свежайшая по сортировке имени.
- **Автораспознавание формата** по корню `Файл`: реестр — из `@ТипИнф`
  (`ЕГРЮЛ_*` / `ЕГРИП_*`), версия — из `@ВерсФорм`. Поддержанные пары —
  `SUPPORTED_FORMATS = {(ЕГРЮЛ,4.08),(ЕГРИП,4.07)}`; неизвестная версия → ValueError.
- **Парсер** `egrn_parser/parsers/egrul_egrip_parser.py` → нормализованная запись
  (одна на `Документ`), единая по форме для будущих источников (checko/dadata/PDF):
  - ЕГРЮЛ: субъект (ОГРН/ИНН/КПП/ОПФ/наим/статус/ОКВЭД) + связи —
    `directors` (ЕИО-физлица), `managing_orgs` (управляющая орг.),
    `founders` (УчрЮЛРос/УчрЮЛИн/УчрФЛ + доля %/номинал), `predecessors`/`successors`
    (реорганизация);
  - ЕГРИП: субъект-физлицо (ОГРНИП/ИННФЛ/ФИО/вид/статус/ОКВЭД).
- **XSD-валидация** через lxml (опциональна, не падает без lxml) — проверена
  на реальных схемах ФНС в cp1251.
- **Тесты** `tests/test_egrul_egrip_parser.py` (9/9) + cp1251-фикстуры
  `tests/fixtures/fns/` (без ПД).

## Решения
- **Нормализованная запись, не запись в БД.** Целевой слой subjects/relations
  проектируется в соседнем чате (`contracts/db/SCHEMA_SPEC.md`) и здесь его нет —
  парсер отдаёт нормализованный dict; враппер «запись→БД» добавится, когда
  таблицы будут готовы (без спекулятивных абстракций).
- **Форма записи единая для мультиисточника** — checko/dadata JSON и PDF
  лягут в ту же структуру `{subject, directors, managing_orgs, founders,
  predecessors, successors, source}`.
