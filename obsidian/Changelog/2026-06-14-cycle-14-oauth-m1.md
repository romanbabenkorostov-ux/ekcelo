## Решения

- **PyJWT вместо authlib.** Roadmap упоминал authlib, но для M1 (только верификация
  Bearer) PyJWT проще, меньше транзитивных зависимостей и лучше документирован.
  authlib может прийти в M2 для browser code-flow если потребуется. Изменение
  не влияет на public API.
- **Opt-in extra `[orchestrator-oauth]`.** Без extras модуль oauth.py
  импортируется, но `OAuthMiddleware` падает при использовании с ясной ошибкой.
  Это даёт backward-compat: deploy без extras + без OIDC env = старый Basic Auth.
- **Стратегия OIDC > Basic > none.** Если задан и OIDC, и Basic — побеждает OIDC
  (явная конфигурация). Никаких неожиданных оверрайдов.
- **JWKS как dict|URL|callable.** dict — для тестов (без HTTP), URL — для prod,
  callable — для динамической ротации. TTL-кеш (10 мин default) спасает от
  частых HTTP-запросов в IdP.
- **`Subject` в `request.state`**, не в `request.user`. Starlette имеет
  built-in AuthenticationBackend (через `request.user`), но он завязан на
  старый паттерн Starlette auth. Свой Subject проще, без зависимости от
  Starlette-внутренностей.
- **HS256 mode для теста/dev.** Прод-OIDC использует RS256 + JWKS, но HS256
  с shared secret полезен для local dev (без поднятия IdP). Активируется когда
  `algorithms=("HS256",)` + `hmac_secret` передан в middleware/verify_jwt.
- **Roles claim как dotted-path.** `roles` (default) для большинства IdP,
  `realm_access.roles` для Keycloak. Без чтения регистра доступа — это
  cycle 15 (RBAC).
