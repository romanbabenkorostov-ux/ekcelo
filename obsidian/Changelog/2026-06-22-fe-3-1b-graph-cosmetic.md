# 2026-06-22 — FE-3.1b: косметика графа (Yandex-KML)

## Проблема
После загрузки Yandex-KML карточка объекта открывается, но под картой —
«волосатый» артефакт: один кружок в центре + смазанный текст сверху, граф не
тянется мышкой.

## Причина
`kmzToGraph` строит узлы из ВСЕХ placemark'ов Yandex-KML. У них нет
`ExtendedData` → `objectType=""` → `kind="doc"` → `layerOf=0`, а радиус слоя 0
= 0 → все узлы схлопываются в одну точку (центр). Рёбер нет (нет `parent_cad`
/`bu_id`). Граф для голой геометрии Yandex бессмысленен. «Не тянется» — у
SVG-графа (FE-2) pan/zoom не предусмотрен by design.

## Что сделал (A+B, мелкий коммит)
- **A.** `renderGraphSvg`: не рисуем граф без связей. Если `nodes ≤ 1` или нет
  валидных рёбер — показываем подпись «Связи владения не заданы (источник без
  структуры собственности)» вместо схлопнутого «супа». Пустой граф (0 узлов)
  как раньше → «Граф пуст».
- **B.** `descriptionLabel` срезает любые HTML-теги (`<[^>]*>`), не только
  `<br/>` — чище метки в списке объектов / графе / попапах карты.

## Не делал (вне рамок)
- Pan/zoom графа — это FE-4 (отдельная задача, нужен когда граф большой в
  api-режиме). KMZ-источник связей владения не несёт, тянуть нечего.
