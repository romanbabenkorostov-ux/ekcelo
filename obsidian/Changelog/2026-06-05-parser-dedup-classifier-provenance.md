- **Дедуп входа PDF/XML** (`cli.py`): один отчёт в `.pdf` и `.xml` → берём **XML**

  значение (фикс шторма diff'ов по `parent_cad_number`). + JSON-массивы сравниваются
  канонически (порядок не важен).
- **Промпты** (`interactive.py`): `Д/н → y/n` с дефолтом из рекомендации (Enter);
  общий выбор `[e/r/n/s/d/q]`, Enter=enrich; фикс `д→d`.

## Ответы на вопросы
- «Провенанс полей»: у поля есть источник (сырой парс vs алгоритм-обогащение). `parent_cad_number`
  ставит обогатитель `resolve_room_parent`; сырая выписка его не содержит. Раньше differ
  считал «было 298 / стало пусто» изменением. Теперь new=None → не изменение.
- «Новый парсинг не видит ранее спарсенное?» — видит: БД парсера (`output/egrn.db`)
  персистентна; diff'ы возникали именно потому, что она помнит прошлый прогон. Чистый
  прогон — удалить `output/egrn.db`.
