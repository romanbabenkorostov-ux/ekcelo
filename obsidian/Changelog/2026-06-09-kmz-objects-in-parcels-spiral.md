# 2026-06-09 — KMZ объектов в пределах ЗУ: контуры + раскладка по спирали

## Ментальный разбор задачи (что было / чего не хватало)
Задача: KMZ для объектов в пределах ЗУ (23:15:0000000:2267, 23:15:0303000:1562,
23:15:0303000:1130); с контурами — наносить контуры, без — точки по спирали внутри ЗУ.
- **Было:** билдер 08 (`pirushin…08_build_kmz`) рисует Point/Polygon/MultiPolygon из
  папки проекта; есть `land_contours` (геометрия).
- **Не хватало (сделано):** (1) механизм «список КН ЗУ → KMZ» из БД; (2) **раскладка
  точек по спирали** внутри полигона ЗУ для объектов без геометрии; (3) KMZ-писатель
  без зависимости `simplekml` (её нет в окружении).

## Реализовано — `egrn_parser/geo_kmz.py`
- `spiral_points(ring, count)` — архимедова спираль внутри полигона ЗУ (радиус
  ∝√i — равномерно по площади; точки вне полигона подтягиваются внутрь). Все точки
  гарантированно внутри границы участка.
- `build_kml/build_kmz(parcels)` — по каждому ЗУ: граница (Polygon), объекты с
  геометрией → контур/точка, объекты без геометрии → точки по спирали внутри ЗУ.
  KMZ = zip с `doc.kml` (без внешних зависимостей). Стили: ЗУ/контур-объект/точка/спираль.
- `collect_from_db(conn, cads)` — граница ЗУ из `land_contours`; объекты внутри из
  `linked_objects`(located_on)/`building_objects.parent`; геометрия объекта — из его
  контуров.
- **Граничный случай:** у ЗУ нет геометрии → спираль ставить негде, объект помечается
  (нужно сначала загрузить контуры).

## CLI
`egrn-parser kmz --parcels "23:15:..,23:15:..,23:15:.." --db <бд> --out objects.kmz`
(13-я команда). Печатает статистику + предупреждение про ЗУ без геометрии.
