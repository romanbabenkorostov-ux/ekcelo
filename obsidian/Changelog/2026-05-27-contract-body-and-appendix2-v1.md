# 2026-05-27 — Рамочный договор: разбивка на скрипты «тело» + «Прил.2»

## Контекст
Договор для ekcelo состоит из 3 блоков:
- **тело** (преамбула, разделы 1–9, реквизиты, подписи);
- **Прил. №1** — ТЗ-1, список объектов (уже отдельный скрипт `10_make_tz1_for_contract_ekcelo_v1.py`, версия v11);
- **Прил. №2** — Календарный план + Перечень источников + Чеклист услуг с ИТОГО.

Поскольку требования к каждому блоку проще тестировать изолированно, на этом
этапе разрабатываем «тело» и «Прил. №2» как **отдельные** скрипты. После
проверки качества формулировок и расчётов планируется собрать единый wizard,
который вызывает три генератора по очереди.

## Что сделано

### `parser/scripts/pirushin_sosn_rocha_11_make_contract_body_v1.py`
Интерактивный CLI генерирует DOCX + MD «Договор № …». Особенности:
- интерактивный выбор Исполнителя/Заказчика из `parties.json`
  (с фоллбэком на ручной ввод или плейсхолдеры «___»);
- два варианта формулировок п. 1:
  - **ГК-39** — возмездные услуги по ст. 779–783 ГК РФ;
  - **135-ФЗ-маппинг** — рамочный договор с указанием, что результаты
    являются исходными данными для последующего отчёта по 135-ФЗ
    (отдельный договор оценки);
- разделы 1–8 + перечень обязательных приложений + раздел 9 «Реквизиты
  сторон» (2-кол. таблица без рамок) + блок подписей;
- автогенерация даты на русском («27 мая 2026г.») и эвристический подбор
  города из адреса Исполнителя;
- DOCX через `python-docx`, MD — параллельно (для diff/review без Word).

### `parser/scripts/pirushin_sosn_rocha_12_make_contract_appendix2_v1.py`
Интерактивный CLI генерирует DOCX + MD «Приложение №2». Содержит три
таблицы:
1. **Календарный план** — 3 этапа (плейсхолдеры под доработку в Word).
2. **Перечень источников информации** — 6 строк (плейсхолдеры).
3. **Чеклист услуг** — 17 строк по утверждённому списку Заказчика;
   цена за ед. × кол-во = сумма; внизу слитая строка «ИТОГО».

Цены и количества читаются из `services_pricing.json` (если есть), иначе
все суммовые ячейки = «___», ИТОГО = «___».

### `obsidian/Database/parties.example.json`
Шаблон файла реквизитов с двумя массивами — `executors` и `clients`.
Скрипт ищет `parties.json` в:
  1. `<папка проекта>/_data/parties.json`,
  2. `<CWD>/parties.json`,
  3. `<repo>/parser/_data/parties.json`,
  4. `<repo>/obsidian/Database/parties.json`.
