# Тестирование базы данных C2 — инструкция для экономиста

> Цель: своими руками собрать БД EKCELO нового поколения (C2), залить в неё данные
> из БД парсера и убедиться, что объекты, права и граф собрались правильно. Технических
> знаний не требуется — повторяй команды по шагам в PowerShell. Время: ~15 минут.

## 0. Что понадобится
- Установленный **Python 3.11+** (проверь: `python --version`).
- Папка проекта: `E:\Code\ekcelo\code` (где лежат `contracts`, `obsidian`, `parser`).
- Файл БД парсера (Block-2) — это `*.db`, который создаёт `egrn_parser` после разбора
  выписок (обычно в выходной папке парсера). Запомни путь к нему.

## 1. Открыть терминал в проекте
В VSCode: `Terminal → New Terminal`. Убедись, что приглашение показывает
`E:\Code\ekcelo\code`. Если нет:
```powershell
cd E:\Code\ekcelo\code
```

## 2. Поставить библиотеки (один раз)
```powershell
pip install "sqlalchemy>=2.0" alembic pytest
```

## 3. Задать, где будет лежать БД C2
Одной командой (путь абсолютный — чтобы все шаги работали с одной БД):
```powershell
$env:EKCELO_DB_URL = "sqlite:///E:/Code/ekcelo/code/contracts/db/ekcelo.db"
```
> Эту строку нужно повторять, если закроешь терминал.

## 4. Создать пустую БД C2 (все таблицы)
```powershell
cd contracts\db
alembic upgrade head
cd ..\..
```
Ожидаемо: в логе три строки `Running upgrade ... 0001 ... 0002 ... 0003`. В папке
`contracts\db` появится файл `ekcelo.db`. Это **пустая** БД из 33 таблиц.

## 5. Залить данные из БД парсера
Подставь путь к своему файлу парсера вместо `<ПУТЬ>`:
```powershell
python -m contracts.db.import_block2 "<ПУТЬ>\parser_block2.db"
```
Ожидаемо — строка вида:
```
imported: {'objects': N, 'entities': M, 'geometries': G, 'subjects': S, 'relations': R, 'assertions': R}
```
Числа должны быть **больше нуля** и примерно соответствовать твоим выпискам
(сколько объектов разобрал парсер — столько и `objects`).

## 6. Построить граф для вьювера
```powershell
python -m contracts.db.graph_emit contracts\db\out\graph.json
```
Ожидаемо: `graph.json: X nodes, Y edges → contracts\db\out\graph.json`.
Файл `graph.json` можно открыть в вьювере проекта.

## 7. Что проверить (приёмка)
Открой `contracts\db\out\graph.json` в любом текстовом редакторе и убедись:
- **`nodesByKind`** в конце файла перечисляет типы: `land`, `building`, `room`,
  `accessory`, `beneficiary_legal` — и их количества похожи на правду.
- В разделе **`edges`** есть рёбра:
  - `owns` / `leases` — кто чем владеет/арендует (из выписок ЕГРН), у них
    **`confidence: 1.0`** (право подтверждено ЕГРН — высшая достоверность);
  - `contains` — что в чём расположено (ЗУ→здание→помещение→принадлежность);
  - `controls` — корпоративные цепочки (если в парсере были), достоверность ниже.

## 8. Автотест (необязательно, но полезно)
```powershell
$env:PYTHONPATH = "E:\Code\ekcelo\code"
pytest contracts\db\tests -q
```
Ожидаемо: `2 passed`.

## 9. Что сказать команде по итогу
Сообщи: (а) числа из шага 5 (`imported: …`); (б) совпало ли число `objects` с
ожидаемым по выпискам; (в) есть ли в графе `owns`-рёбра с `confidence 1.0`;
(г) любые ошибки красным текстом — скопируй их целиком.
