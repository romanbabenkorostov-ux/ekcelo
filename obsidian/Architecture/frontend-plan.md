# План фронтенда Ekcelo + удобство без фронтенда

> Сейчас система — CLI + Excel/текстовые шаблоны + Obsidian-документы. Это рабочий
> вариант для одного оператора. Ниже: (А) как удобнее уже сейчас без фронтенда,
> (Б) варианты фронтенда и фазовый план.

## А. Удобнее без фронтенда (доступно сейчас)
1. **Папка проекта = шаблоны.** Экономист кладёт заполненные файлы из
   `Golden-Path-Economist/templates/` в папку проекта (геометрия .json, перечень .txt,
   техкарта .xlsx, ОСВ .xlsx, правки .yaml, лот .yaml).
2. **Один прогон конвейера.** Демо `parser/scripts/golden_path_demo.py` показывает весь
   путь; на его основе — команда «прогнать папку проекта» (см. фазу 0 ниже).
3. **Результаты — в БД + Bundle.** `egrn-parser bundle --export-c2` собирает пакет;
   оценочные данные смотреть запросами к БД (вьюхи `v_vineyard_valuation` и пр.).
4. **Obsidian как «панель».** Документы/индексы шаблонов, ADR, changelog — уже в
   хранилище; навигация по ссылкам `[[…]]`.

**Фаза 0 (мини, без UI):** CLI-команда `egrn-parser ingest-project --dir <папка>` —
автоопределяет типы файлов в папке и грузит всё за один вызов (обёртка над уже
готовыми парсерами). Это закрывает 80% удобства до появления фронтенда.

## Б. Варианты фронтенда (нужно ваше решение)
| Вариант | Что это | Плюсы | Минусы |
|---|---|---|---|
| **1. Веб (FastAPI + браузер)** | формы загрузки + просмотр результатов; уже есть `egrn-parser serve` (FastAPI) | стандартно, доступ с любого ПК, переиспользуем backend | нужен веб-разработчик, развёртывание |
| **2. Obsidian-нативный** | кнопки/формы прямо в вашем хранилище (Templater/Buttons/Dataview) | вы уже в Obsidian, минимум новой технологии | ограничен Obsidian, не «настоящее» приложение |
| **3. Десктоп-приложение** | локальная программа (окна, drag-drop файлов) | работает офлайн на Win10, без браузера | отдельная сборка/обновления |
| **4. Только CLI + Excel** | оставить как есть, отполировать шаблоны/команды | дёшево, уже работает | требует запуска команд вручную |

## Фазовый план (общий, уточняется под выбранный вариант)
- **Фаза 1 — Загрузка:** экран/форма «папка проекта → загрузить» (типы файлов,
  отчёт что распозналось, ошибки валидации).
- **Фаза 2 — Просмотр:** оценочный профиль виноградника, состав лота, ЭТП-профиль
  объекта (с пометками источника/уверенности), реестр ОС.
- **Фаза 3 — Сборка пакета:** кнопка «Собрать Bundle (C2)» + выгрузка.
- **Фаза 4 — Карта:** показ контуров/геоточек (KMZ/Leaflet) — связь с C1 (KMZ).

## Рекомендация
Для одного-двух операторов-экономистов: **сначала Фаза 0** (команда `ingest-project`,
1-2 дня), затем — **вариант по вашему выбору** (ниже задан вопрос). Веб (1) — если
нужен доступ нескольким людям/с разных ПК; Obsidian (2) — если хотите остаться в
текущем инструменте с минимумом нового.
