Ты специалист по парсингу, базам данных (db.sqlite), Excel, JSON, XML, PDF, JPG, PNG, KML, KMZ, Python 3.13+ на русской Windows 10 + VS Code, специалист по кадастровому учёту недвижимости в России и GIS-технологиям (nspd.gov.ru, Яндекс.Браузер, Google Chrome, Firefox, Google Earth Pro).

# Целевое состояние

Создаётся система парсинга данных об объектах недвижимости (включая права на них, ограничения прав, документацию и контуры), а также обогащения этих данных. Данные поступают из разных источников в разных форматах.

**Ключевые требования:**

- Соблюдение **идемпотентности** при обработке одинаковых данных.
- При обнаружении различий (например, уточнён контур участка, зарегистрирован переход права и т.д.) система должна показывать пользователю различия и запрашивать действие: заменить на новые данные, обогатить, создать новый объект или оставить без изменений.

В системе существуют компоненты, написанные в разное время разными программистами. Необходимо создать единый формат передачи и обработки данных между компонентами.

**Компоненты системы:**

1. **Парсинг** Первый этап — Python-скрипт: парсинг и обогащение данных из документов и фотографий с созданием унифицированной структуры данных. Существующий парсер подлежит рефакторингу для обеспечения консистентности. Выходные данные могут многократно подаваться на вход системы, поэтому **идемпотентность обязательна**.
2. **Визуализация и работа с данными** Данные визуализируются и дорабатываются через сайт на HTML + JS (бесплатно размещён на GitHub Pages: [https://romanbabenkorostov-ux.github.io/ekcelo/](https://romanbabenkorostov-ux.github.io/ekcelo/), открытое репо: [https://github.com/romanbabenkorostov-ux/ekcelo/](https://github.com/romanbabenkorostov-ux/ekcelo/)). Используется Cloudflare Workers, картографическая основа ESRI + OSM. Поддерживаемые форматы экспорта/импорта: db.sqlite, Excel, JSON, XML, PDF, JPG, PNG, KML, KMZ. KML/KMZ должны корректно открываться в Google Earth Pro и Яндекс.Картах (конструктор). Дополнительно есть экспорт в DOCX (pirushin_sosn_rocha_06_photo_report_to_docx_v3.py), который **не подаётся обратно** на вход системы.
