# Шаблоны входных данных Ekcelo + описания

Заполните нужные шаблоны своими данными и загрузите. К каждому шаблону — файл-описание
(колонки/поля, как загрузить). Не нужно заполнять все — только то, что есть по объекту.

| Что загружаете | Шаблон | Описание | Куда в БД |
|---|---|---|---|
| Геометрия участка (NSPD) | [[nspd_geometriya_TEMPLATE.json]] | [[nspd_geometriya.ОПИСАНИЕ]] | `land_contours` |
| Перечень виноградников (залог) | [[perechen_nasazhdeniy_TEMPLATE.txt]] | [[perechen_nasazhdeniy.ОПИСАНИЕ]] | `agro_parcel`/`agro_crop_cycle` |
| Техкарта виноградника | `tehkarta_vinogradnik_TEMPLATE.xlsx` | [[tehkarta_vinogradnik.ОПИСАНИЕ]] | `agro_event`/… |
| ОСВ — техника / ОС | `osv_tehnika_TEMPLATE.xlsx` | [[osv_tehnika.ОПИСАНИЕ]] | `fixed_asset` |
| Ваши правки (ЭТП) | [[etp_pravki_TEMPLATE.yaml]] | [[etp_pravki.ОПИСАНИЕ]] | `object_etp_profile` |
| Определение лота | [[lot_TEMPLATE.yaml]] | [[lot.ОПИСАНИЕ]] | `lots`/`lot_items` |

> Excel-шаблоны (`.xlsx`) Obsidian не показывает в превью — открывайте в Excel
> (правый клик → «Показать в проводнике»/Open in default app).

## Все шаблоны проверены парсерами ✅
- ОСВ → 3 ОС (в т.ч. 1 ОКС на счёте 01.08);
- Техкарта → виноград, операции + пестициды/удобрения;
- Перечень/геометрия/YAML — совместимы с загрузчиками.

## Порядок загрузки (по золотому пути)
1. Геометрия → контуры земли. 2. Перечень насаждений. 3. Техкарта. 4. ОСВ (техника).
5. Ваши правки (ЭТП). 6. Лот. 7. Сборка Bundle (`egrn-parser bundle --export-c2`).
Демо всего пути на мок-данных: `parser/scripts/golden_path_demo.py`.
