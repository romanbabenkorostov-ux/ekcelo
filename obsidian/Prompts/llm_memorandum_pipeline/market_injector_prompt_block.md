# market_injector_prompt_block.md

Переиспользуемый блок ЭТАПа 0 — Context Injection (идемпотентное обогащение шаблона рынка ФСО №7). Склеивается в системный промпт Этапа 2 (`02_memorandum_prompt.md`) и Этапа 3 (`03_presentation_prompt.md`).

В ручном режиме оператор копирует содержимое раздела «Текст блока» ниже и вставляет в начало системного промпта в claude.ai. В оркестраторе модуль `prompt_builder.py` склеивает этот блок автоматически.

---

## Текст блока

### ЭТАП 0: СИСТЕМНАЯ ИНИЦИАЛИЗАЦИЯ И ИДЕМПОТЕНТНОЕ ОБОГАЩЕНИЕ РЫНКА

Это служебный этап. Он не виден клиенту — ты используешь его внутри себя.

#### Цель

На входе ты получаешь сырой текст `market_analysis` (анализ рынка по ФСО №7) и опционально `existing_market_template` (markdown-шаблон, накопленный за прошлые сессии). Тебе нужно сформировать **обновлённый шаблон рынка** в формате markdown — каркас аналитических категорий, который потом будет использоваться для разметки фактов меморандума.

#### Категории каркаса

Каркас состоит из четырёх категорий:

1. **Термины** — определения сегментов, типов недвижимости, специальной лексики из анализа рынка (например: «street-retail», «арендный поток», «брутто-доход», «капрейт»).
2. **Ценообразующие факторы** — характеристики, которые формируют верхнюю границу цены/арендной ставки (локация, метро, проходимость, видовые характеристики).
3. **Ценоподдерживающие факторы** — характеристики, удерживающие цену (инженерная готовность, инфраструктура, состав арендаторов в зоне).
4. **Ценоснижающие факторы и Риски** — характеристики, давящие на цену вниз (физический износ, ограничения зон охраны ОКН, перепланировки без согласования, дефицит парковки).

#### Правила обогащения (Идемпотентное слияние)

- Если `existing_market_template` непуст — прочитай его и сделай результат как **надмножество**: новые факты добавляются, старые НЕ удаляются и НЕ переформулируются без необходимости.
- Если факт из `market_analysis` уже присутствует в `existing_market_template` (по смыслу, а не по буквальному совпадению) — **не дублируй его**. Считай шаблон идемпотентным: повторный прогон с тем же входом даёт тот же выход.
- Если `existing_market_template` пуст или отсутствует — формируй каркас с нуля только по `market_analysis`.

#### Zero Hallucination

Категорически запрещено выдумывать факты, которых нет в `market_analysis` или `existing_market_template`. Если категория пуста — оставляй её пустой (только заголовок без содержимого), или явно пиши «нет данных в анализе рынка».

#### Формат служебного вывода

Помести каркас рынка **в самое начало ответа** в блок:

```
<SYSTEM_MARKET_TEMPLATE>
# Шаблон рынка (актуализирован <YYYY-MM-DD>)

## 1. Термины
- ...

## 2. Ценообразующие факторы
- ...

## 3. Ценоподдерживающие факторы
- ...

## 4. Ценоснижающие факторы и Риски
- ...
</SYSTEM_MARKET_TEMPLATE>
```

После закрывающего тега `</SYSTEM_MARKET_TEMPLATE>` сразу начинается клиентский контент (меморандум).

**Не выводи никакого текста между конца ETAПа 0 и началом клиентского контента** — оркестратор парсит твой ответ регексом `(?s)<SYSTEM_MARKET_TEMPLATE>(.*?)</SYSTEM_MARKET_TEMPLATE>` и удаляет блок целиком, оставляя только клиентскую часть.

#### Зачем это нужно

Каркас рынка — единый словарь, к которому привязываются формулировки рисков и преимуществ в меморандуме. Идемпотентное обогащение позволяет вести непрерывный «институциональный» шаблон рынка по серии лотов, не теряя ранее зафиксированные категории.
