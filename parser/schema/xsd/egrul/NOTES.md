# XSD ФНС — ЕГРЮЛ (сведения о юридических лицах)

## Назначение

`egrn_parser/parsers/egrul_egrip_parser.py` парсит и (через lxml) валидирует
XML-выписки ЕГРЮЛ по XSD из этой папки.

Парсер опознаёт формат по корню `Файл`:
- `@ТипИнф` начинается на `ЕГРЮЛ_` → реестр ЕГРЮЛ (`ЕГРЮЛ_ОТКР_СВЕД` /
  `ЕГРЮЛ_ЗАКР_СВЕД` / `ЕГРЮЛ_ОГР_СВЕД`);
- `@ВерсФорм` → версия формата (текущая — `4.08`).

`find_xsd("ЕГРЮЛ", версия)` берёт **самый свежий** файл по сортировке имени
(если в имени есть подстрока версии — предпочитает его).

## Источник

XSD публикует ФНС (ГНИВЦ) в составе требований к составу и структуре файла,
содержащего сведения из ЕГРЮЛ. Файл — в кодировке **windows-1251**, без
targetNamespace (обход по «голым» локальным тегам).

- Реестр форматов ФНС: https://www.nalog.gov.ru/
- Текущая редакция в репо: `VO_RUGF_2_311_26_04_08_01.xsd` — версия формата **4.08**.

## Обновление редакции

1. Скачать новый `VO_RUGF_*.xsd`.
2. Положить **рядом** со старым (не заменять — старые нужны для архивных выписок).
3. Добавить новую пару `("ЕГРЮЛ", "<версия>")` в `SUPPORTED_FORMATS`
   (`egrul_egrip_parser.py`) и проверить, что новые/переименованные теги
   разбираются корректно.
4. Отметить редакцию в `obsidian/Changelog/`.
