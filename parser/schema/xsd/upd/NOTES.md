# XSD ФНС для УПД (универсальный передаточный документ)

## Назначение

`parser/upd/validator.py` валидирует УПД-XML по XSD ФНС из этой папки.
Берёт самый свежий файл по маске `ON_NSCHFDOPPR_*.xsd` (sorted →
последний — самая новая редакция).

## Источник

XSD публикует ФНС в составе пакета «Электронные форматы документов»
(КНД 1115131 — счёт-фактура / УПД).

  • Реестр: https://www.nalog.gov.ru/rn77/related_activities/el_format/sf/
  • Машиночитаемая XSD-схема прилагается ФНС вместе с приказом
    об утверждении конкретной редакции (например, приказ
    ЕД-7-26/970@ от 19.12.2023 — действующая редакция).

## Содержимое папки

При первом коммите (Step 2 PR) этой папки в репо XSD-файл может
отсутствовать — пользователю предложено приложить
`ON_NSCHFDOPPR_1_997_01_05_03_05.xsd` (или более свежую редакцию) в
отдельном ZIP. Без XSD `parser.upd.cli validate` возвращает single
ошибку «XSD не найден» (см. `validator._find_xsd`).

## Обновление редакции

При выходе новой редакции ФНС:

1. Скачать новый `ON_NSCHFDOPPR_<новая_версия>.xsd`.
2. Положить в эту папку **рядом** со старыми (не заменять — старые
   нужны для валидации архивных документов).
3. Обновить дату редакции в Changelog.

Validator всегда берёт **самый свежий** файл по сортировке имени.
