# Миграция со скриптов pirushin_sosn_rocha_* → egrn_parser v1.10

## Соответствие старых скриптов новым командам

| Старый скрипт | Новая команда |
|---------------|---------------|
| `pirushin_sosn_rocha_01_*.py` | `egrn-parser parse --input <папка>` |
| `pirushin_sosn_rocha_02_*.py` | `egrn-parser folders create --root <папка> --cad-file <файл>` |
| `pirushin_sosn_rocha_03_*.py` | `egrn-parser folders distribute --root <папка> --source <входящие>` |
| `pirushin_sosn_rocha_04_*.py` | `egrn-parser folders validate --root <папка>` |
| `pirushin_sosn_rocha_05_*.py` | `egrn-parser parse ... && egrn-parser export --xlsx <файл>` |
| `pirushin_sosn_rocha_06_*.py` | Остаётся отдельным скриптом (финальный фотоотчёт, не подаётся обратно) |

## Пошаговый переход

1. **Установить пакет:**
   ```bash
   pip install -e .
   ```

2. **Создать новую БД:**
   ```bash
   egrn-parser migrate --db ./output/egrn.db
   egrn-parser dict-load --db ./output/egrn.db
   ```

3. **Разобрать существующие выписки:**
   ```bash
   egrn-parser parse --input ./ОБЪЕКТЫ --db ./output/egrn.db
   ```

4. **Экспортировать результаты:**
   ```bash
   egrn-parser export --db ./output/egrn.db \
     --xlsx ./output/egrn.xlsx \
     --graph-json ./output/graph.json \
     --xlsx-template ./templates/2026_02_04_Assets_Импорт_Объект2.xlsx
   ```

## Важные изменения v1.10

### Обременения / ограничения прав

**Было (v1.9):** текст обременений склеивался через ` | ` в поле `right_restrictions`.

**Стало (v1.10):** три категории в `rights.right_category`:
- `'right'` — зарегистрированные права собственности/аренды
- `'encumbrance'` — обременения с бенефициаром (ипотека, аренда, сервитут)
- `'restriction'` — ограничения прав без бенефициара (запрет, арест)

### Ограничения объекта

**Стало (v1.10):** хранятся как JSON-массив в `land_objects.object_restrictions` и
`building_objects.object_restrictions`. **Не в таблице rights.**

### Этажность зданий

Новые поля: `floors_total`, `floors_above_ground`, `underground_floors`,
`floors_inspection`, `condition_inspection`.

### Помещения → этажность родительского здания

Алгоритм `resolve_room_parent()` автоматически подтягивает этажность здания
к записям помещений (`parent_floors_above_ground`, `parent_underground_floors`).

## Откат

```bash
egrn-parser migrate --db ./output/egrn.db --rollback
```
