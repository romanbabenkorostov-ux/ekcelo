# C5 — Lot model (лот)

> Лот — сборка по **дополняющим и исключающим признакам**, в которой соединяется
> ветка **физических активов** (недвижимость, основные средства) с веткой
> **имеющихся прав**, с указанием **точки актуальности** (as-of date).

## Определение

Лот = именованный набор объектов (КН + оборудование), отобранный по правилам
include/exclude на дату актуальности. Хранится в §6 БД: `lots` / `lot_items`
(ADR-001, ручной не-ЕГРН слой; при пересоздании БД из выписок не восстанавливается).

## Признаки сборки

```json
{
  "lot_id": "rostov/suvorova-52/2026Q2",
  "as_of_date": "2026-06-03",
  "include": {
    "addresses": ["Ростов-на-Дону, Суворова, 52"],
    "beneficiaries_inn": ["6164000000"],
    "cad_numbers": ["61:44:0050706:31"],
    "with_equipment": true
  },
  "exclude": {
    "cad_numbers": ["61:44:0050706:99"],
    "with_equipment": false
  },
  "members": ["61:44:0050706:31", "..."]
}
```

- **include** — дополняющие признаки (адреса / бенефициары / КН / наличие оборудования).
- **exclude** — исключающие признаки (вычитаются из include-множества).
- **as_of_date** — точка актуальности: какие права/выписки/состояние берём.
- **members** — материализованный результат отбора (детерминированный).

## Две ветки, сходящиеся в лоте

| Ветка | Источник | Узлы графа (C4/C1) |
|-------|----------|--------------------|
| Физические активы | ЕГРН §1 (ЗУ/ОКС/помещения/сооружения/ОНЗ) + ОСВ (оборудование) | `land/building/room/structure/ons/equipment/level` |
| Имеющиеся права | ЕГРН §3 права/§5 обременения + ЕГРЮЛ/ЕГРИП бенефициары | `right/encumbrance/beneficiary_*` |

Связка веток — по `cad_number` (якорь) и `graph_node_id`.

## Маппинг на БД §6

- `lots(lot_id, name, as_of_date, include_json, exclude_json, primary_cad_number, …)`
- `lot_items(lot_id, cad_number, role, ord)` — role ∈ {building, land, room, equipment, structure}.

## Идемпотентность

Один и тот же `include/exclude/as_of_date` на одной БД → один и тот же `members[]`
(детерминированный отбор). Лот экспортируется как Bundle `kind=lot` (C3).
