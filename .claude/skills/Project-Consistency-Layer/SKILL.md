## Основные правила (всегда соблюдать)

### 1. Единый источник правды
- `schema/current_schema.sql` — главная схема БД
- `CLAUDE.md` — основные правила поведения
- `SUMMARY.md` — текущее состояние проекта
- `docs/db-mapping.md` — сопоставление данных парсера и просмотрщика

### 2. Структура репозитория
- `parser/` — весь Python-код парсера
- `/` (корень) — HTML/JS просмотрщик + Cloudflare Worker
- `schema/` — миграции и текущая схема
- `decisions/` — важные архитектурные решения
- Obsidian: `E:\Code\ekcelo\obsidian_ekcelo\` — основная база знаний

### 3. Работа с БД
- Все изменения схемы только через миграции
- Названия полей и таблиц — строго `snake_case`
- Python и JS должны использовать одинаковые названия полей
- Перед любым изменением БД — сверяться с `current_schema.sql`

### 4. Multi-Subscription (несколько подписок)
- Никогда не полагайся на память чата
- При начале работы всегда предлагай `git pull`
- Перед окончанием — обновлять `SUMMARY.md` и нужные файлы
- Документировать все важные решения в `decisions/`

### 5. Obsidian Sync
- При важных изменениях схемы, архитектуры или логики — предлагать создать/обновить заметку в Obsidian
- Структура заметок: Database/, Parser/, Frontend/, Architecture/, Decisions/

### 6. Token Economy + Karpathy Style
- По умолчанию работать в кратком стиле (Caveman full)
- Быть хирургически точным в изменениях
- Думать перед кодом, показывать trade-off'ы только когда важно
- Предлагать короткий или подробный режим по запросу

## Активация
Этот skill активируется автоматически при открытии проекта.
