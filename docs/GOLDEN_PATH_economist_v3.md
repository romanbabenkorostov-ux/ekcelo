# Золотой путь обработки данных (v3)

> **Изменения v3 относительно v2:**
> - Дерево проекта переработано: убраны цифровые префиксы (`00_…07_`),
>   корневые папки сгруппированы по назначению: `ОСВ/`, `XLSX/`, `DB/`,
>   `Документы_JPG/`, `Выписки_PDF/`, `Фотографии/`, `Бизнес_единицы/`,
>   `KMZ-KML/`, `HTML/`, `_data/`.
> - Болванка проекта и идемпотентная синхронизация фото-дерева
>   делаются **одним скриптом** — `pirushin_sosn_rocha_07_init_project_v1.py`
>   (меню 1/2/3 в начале).
> - Конвертация документов в JPG расширена на `.pdf`, `.doc`, `.docx`
>   (последние два — через LibreOffice или MS Word). Имя JPG несёт КН
>   и дату документа.
> - В фото-дереве вместо `По_объектам/<КН>/` теперь
>   `Недвижимость/<категория>/<КН>/`, где категория — одна из:
>   `Земельные_участки / Строения / Сооружения / Помещения / ОНЗ`.
>   У строений/сооружений/ОНЗ автоматически создаётся подпапка `План/`.

## 2. Шаги обработки

### Шаг 0. Создать болванку

```powershell
python pirushin_sosn_rocha_07_init_project_v1.py
```
→ Меню → **1. Создание структуры** →
→ корневая папка (Enter — `D:\ОБЪЕКТЫ`) →
→ имя проекта →
→ дерево создаётся, путь копируется в буфер обмена.

### Шаг 1–4. Сбор источников (как в v2)
Кладите файлы в подпапки:
- ЕГРН-PDF + парные XML  → `Выписки_PDF\ЕГРН\`
- Свидетельства          → `Выписки_PDF\Свидетельства_о_праве\`
- Технические паспорта   → `Выписки_PDF\Технические_паспорта\`
- Техпланы               → `Выписки_PDF\Техпланы\`
- Прочее (.doc, .docx, KML/KMZ, договоры) → `Выписки_PDF\Прочее\`
- Выписки ЕГРЮЛ/ЕГРИП    → `Выписки_PDF\ЕГРЮЛ-ЕГРИП\`
- Оборотно-сальдовая ведомость → `ОСВ\osv.xlsx`

### Шаг 5. Конвертировать документы в JPG (новый)

```powershell
python pirushin_sosn_rocha_07_init_project_v1.py
```
→ Меню → **2. Конвертация документов** → Enter (берёт последний проект).

Скрипт:
- рекурсивно обходит `Выписки_PDF\` (все подпапки),
- конвертирует `.pdf`, `.doc`, `.docx` в JPG (`Документы_JPG\<подпапка>\`),
- именует машиночитаемо (`egrn_61_44_…_d20260427_p01.jpg` и т.п.),
- пишет EXIF: GPS центра объекта, UserComment-JSON (КН, ИНН, ОГРН, дата,
  страница из N, height_m, AMSL),
- идемпотентен: при повторе пропускает уже сделанное;
- если у пары PDF+XML совпадают КН и №КУВИ — данные из XML
  переписывают PDF-извлечения,
- по завершению **синхронизирует фото-дерево**: для каждого
  распознанного КН (из XML-выписок, structure.json, имён JPG, KML/KMZ)
  создаёт подпапку в `Фотографии\Недвижимость\<категория>\<КН>\` и при
  необходимости `План\`.

> .doc/.docx требуют LibreOffice (бесплатно) или MS Word на Windows
> (`pip install comtypes`). Если ни одного нет — .pdf обрабатываются
> штатно, .doc/.docx пропускаются с подсказкой.

### Шаг 6. Сведение источников (`03_enrich_v14`)
Без изменений относительно v2.

### Шаг 7. ОСВ → структура (`052_make_structure_v1`)
Вход теперь — `<root>\ОСВ\osv.xlsx`. Выход — `_data\structure.json`.

### Шаг 8. Граф (`04_nspd_graph_v14`)
Выход — `HTML\graph.html` (в v2 был `_data\graph.html`).

### Шаг 9. KMZ (`08_build_kmz_v1`)
Выход — `KMZ-KML\project.kmz` (идемпотентно перезаписывается на каждый
запуск). Совместим с Google Earth Pro и
`https://romanbabenkorostov-ux.github.io/ekcelo/`.

## 3. Идемпотентность фото-дерева

Можно повторять Шаг 5 хоть после каждого добавления документа:
- новые КН → добавятся подпапки в `Фотографии\Недвижимость\<категория>\`,
- старые → не пересоздаются,
- удалённые из источников КН — **папка остаётся** (это пользовательский
  контент, скрипт не удаляет).

Источники определения категории (в порядке приоритета):
1. `_data\egrn_xml.json` — точный `obj_type` из XML Росреестра.
2. `_data\structure.json` (если уже сделан Шаг 7).
3. EXIF.UserComment.object_type в `Документы_JPG\**\*.jpg` (записывается
   на Шаге 5).
4. Имена JPG-документов `egrn_/svid_/tehpasp_/tehplan_/doc_` — без типа
   (КН попадает в учёт, но папка не создаётся под него).
5. KML/KMZ в `Выписки_PDF\` и `KMZ-KML\` — без типа.

## 5. Частые проблемы

| Симптом | Решение |
|---|---|
| .doc/.docx не конвертируются | Поставь LibreOffice (https://www.libreoffice.org/) или, если на Win есть Word — `pip install comtypes` |
| `structure.json пока не создан` | Это нормально для свежей болванки; GPS добавится после Шага 7 |
| Папка `Фотографии\Недвижимость\<КН>\` не создалась | Тип объекта пока не определён ни из XML, ни из structure.json, ни из EXIF; добавь XML-выписку или прогони Шаг 7 |
| .docx-файл с КН в теле не попал в JPG | Проверь, что LibreOffice/Word установлен. КН ищется и без них (читается прямо из docx) — но JPG не сделать без рендера |
| `[skip] file.pdf (N стр. уже сконвертированы)` | Идемпотентность работает; чтобы переконвертировать — удалите старые JPG в `Документы_JPG\` |
