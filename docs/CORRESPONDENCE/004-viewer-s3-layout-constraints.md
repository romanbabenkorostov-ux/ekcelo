## A. Viewer = атомарный блок (move-together)

`index.html` ссылается **относительно**: `./sw.js` (регистрация SW,
`index.html:1580`) и `./v2961.html` (замороженная версия, `:1107`). Поэтому
**в один каталог и одним движением**:

```
viewer/
├── index.html
├── sw.js
└── v2961.html
```

Тогда `./sw.js` и `./v2961.html` продолжают резолвиться **без правок кода**
(минимальный риск). Разносить эти три файла по разным местам — нельзя.

## B. GitHub Pages deploy (блокер)

`.github/workflows/deploy.yml` сейчас: триггер по `index.html`,
`upload-pages-artifact` `path: "."` (сайт = корень репо, лендинг =
`/index.html`). После переезда корень `/` будет 404.

**Требование (viewer-domain — деплой приложения):** в том же PR обновить
`deploy.yml` так, чтобы **корень сайта отдавал `viewer/index.html`**.
Рекомендуемо: `upload-pages-artifact` `path: "viewer"` (сайт-рут = `viewer/`),
триггер-фильтр `index.html` → `viewer/**`. Тогда SW-scope = `/` и
относительные `./sw.js`/`./v2961.html` работают как есть.

## C. Непрерывность Service Worker

Смена URL `sw.js` осиротит установленный у возвратившихся пользователей SW.
Требование: **bump `CACHE_NAME`** (`ekcelo-cadastre-v2` → `…-v3`); существующий
`activate`-cleanup (удаляет не-текущие кэши) оставить — стухший тайл-кэш
вычистится сам. Регистрация остаётся `./sw.js` (относительная,
самокорректируется при co-location из п.A). Перехват тайлов `nspd.gov.ru`
не зависит от scope (fetch-handler по URL) — регрессии нет.

## D. Статичность

Приложение — один статический `index.html`, **без сборки**. Все
viewer-пути держать относительными (никаких абсолютных `/viewer/...`), чтобы
работало и под Pages-рутом, и под подпутём, и из `file://`
(см. комментарий v2.9.39 про `file://`). Не вводить бандлер в S3.

## E. dev/ и тесты

## F. Классифицировать root-мусор

`worker.js`, `worker_good_work2026-04-26.js` в корне — `index.html` их
**не дёргает** (нет `new Worker(...)`/ссылок). Похоже на легаси. Просьба
подтвердить: если не используются парсером — в S3 архивировать/удалить
(чистка), не переносить вслепую. Если used — ваш домен, ваше решение.
