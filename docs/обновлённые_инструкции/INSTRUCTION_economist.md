# Инструкция для экономиста: запуск скрипта построения структуры

Скрипт превращает **ОСВ по счёту 01 из 1С** в иерархический JSON-файл:
предприятие → бизнес-единицы → кадастровые объекты → оборудование.
Результат используется в дальнейших обработках (обогащение через ЕГРН,
визуализация, БД).

> Предполагается, что у вас уже:
> - установлен **VS Code** и Python-расширение от Microsoft,
> - есть **виртуальное окружение `venv`** для проекта,
> - создана **болванка проекта** скриптом
>   `pirushin_sosn_rocha_07_init_project_v1.py` (меню → 1).

---

## 1. Что подготовить (5 минут до запуска)

### 1.1. Файл ОСВ из 1С

В 1С: **Отчёты → Оборотно-сальдовая ведомость по счёту → счёт 01 →
сформировать → Сохранить как → Лист Excel 2007 (`.xlsx`)**.

Положите файл в папку **`ОСВ/`** внутри болванки проекта.

Файл должен выглядеть так (первые строки):

```
Общество с ограниченной ответственностью "САНАТОРИЙ ..."
Оборотно-сальдовая ведомость по счету 01 за 1 квартал 2026 г.

Счет        Показатели      Сальдо на начало периода   Обороты за период   ...
            БУ              ...
01          БУ              1 761 997 483,75           ...
01.01       БУ              ...
 Торшер 01_1192             89 200
            НУ              89 200
            ВР
 ...
```

**Чего избегать:**
- ❌ файл `.xls` (старый Excel) — конвертируйте в `.xlsx`;
- ❌ файл с включённой группировкой/свёрнутыми разделами — разверните всё перед сохранением;
- ❌ файл, выгруженный «как печатная форма» с верхним и нижним колонтитулами на каждой странице — нужен «обычный» вариант.

### 1.2. (Опционально) предыдущий `structure_*.json`

Если вы **уже запускали** скрипт ранее и хотите дополнить/исправить
данные — найдите файл вида `structure_sanatoriy_..._20260516_073621.json`
в `_data/` рядом с ОСВ. При повторном запуске скрипт спросит путь к нему
и **сохранит ваши подтверждения** (переименования бизнес-единиц, ручные
правки и т.п.).

### 1.3. (Опционально) каталог с выписками ЕГРН/NSPD

Если кадастровый инженер уже выгрузил по объектам JSON-файлы скриптом
`01_parsing_nspd_gemini_v5.py`, проверьте, что они лежат в
`json/objects/` — скрипт сам их подхватит, и группировка бизнес-единиц
станет точнее (используются адреса).

### 1.4. (Опционально) хаотичный «дамп» от вас лично

Если у вас есть произвольная папка с фото, PDF-выписками, договорами
в DOC/XLS, базой `.db`, HTML-отчётами, KML-файлами — **сложите всё в
папку `Не_распределено/` в корне болванки** и запустите:

```
python pirushin_sosn_rocha_07_init_project_v1.py
```
→ **меню → 3. Сортировка папки Не_распределено/**.

Скрипт разнесёт всё по структуре проекта (фото → `Фотографии/Недвижимость/<кат>/<КН>/<подпапки>/`,
PDF → `Выписки_PDF/<подпапка>/`, XLS → `XLSX/`, DB → `DB/`, …), сначала
покажет план (dry-run) и спросит подтверждение. Для JPG допишет в EXIF
привязку к КН и категории, **не затрагивая** оригинальные даты съёмки и
GPS-координаты, и **сохранит исходный mtime файла**. Это можно сделать
как до запуска парсера ОСВ, так и после.

## 2. Запуск скрипта в VS Code

### 2.1. Открыть проект

`Файл → Открыть папку…` → выбрать папку, где лежит файл
`pirushin_sosn_rocha_052_make_structure_v1.py`.

### 2.2. Активировать виртуальное окружение

VS Code обычно подхватывает `venv` автоматически (внизу справа видна
выбранная версия Python). Если нет:

`Ctrl+Shift+P` → **`Python: Select Interpreter`** → выбрать
интерпретатор из `venv` (путь содержит `\venv\Scripts\python.exe`).

### 2.3. Установить зависимости (только один раз)

Открыть терминал (`Ctrl+` `` ` ``) — слева в нём должно отображаться `(venv)`.
Затем:

```powershell
pip install openpyxl
```

### 2.4. Запустить скрипт

В терминале:

```powershell
python pirushin_sosn_rocha_052_make_structure_v1.py
```

Альтернатива: открыть файл скрипта в редакторе, нажать **▶ Run Python File**
в правом верхнем углу.

## 3. Диалог с программой — что отвечать

Скрипт задаст три вопроса, затем перейдёт к интерактивному назначению
бизнес-единиц.

### Вопрос 1: Путь к `.xlsx` ОСВ

Скопируйте полный путь к файлу (в Проводнике: правый клик по файлу →
**«Копировать как путь»**) и вставьте. **Кавычки можно не убирать** —
скрипт их сам обрежет.

Пример:
```
Путь к .xlsx ОСВ счёта 01: "D:\Проекты\Сосновая_Роща\ОСВ\Сосновая_Роща_1кв2026.xlsx"
```

### Вопрос 2: Путь к предыдущему `structure_*.json`

Если повторный запуск — вставьте путь. Если первый — просто нажмите
**Enter**.

### Вопрос 3: Каталог с JSON-выгрузками NSPD/ЕГРН

Если есть папка с выписками от инженера (обычно `json/objects/` в
болванке) — путь к ней. Если нет — **Enter**.

После этого скрипт за несколько секунд распарсит ОСВ и покажет
статистику:

```
⏳ Парсинг ОСВ…
  предприятие: ООО «САНАТОРИЙ "СОСНОВАЯ РОЩА"»
  период:      Оборотно-сальдовая ведомость по счету 01 за 1 квартал 2026 г.
  ОС-записей:  711
    01.01  (собственность ):  651
    01.03  (аренда        ):    8
    01.09  (аренда        ):   44
    01.К   (аренда        ):    8
```

## 4. Фаза А — бизнес-единицы по адресам

Скрипт сгруппирует оборудование, у которого удалось определить адрес
кадастрового объекта (через NSPD-выписки). Для каждой группы:

```
  Адрес: г. Ялта, ул. Парковая, 22
  Объектов ОС в этой группе: 156
  Примеры: Кровать односпальная · Торшер 01_1192 · ...
  Имя бизнес-единицы [г. Ялта, ул. Парковая, 22]:
```

**Что делать:**
- **Enter** — оставить адрес как имя бизнес-единицы.
- **Ввести своё имя** (например, «Корпус Гамма» или «Лечебный корпус») и Enter.

Если NSPD-выписок не было — Фаза А просто пропустится.

---

## 5. Фаза Б — оборудование без адреса

Для остального оборудования (которое не удалось привязать к адресу)
скрипт сначала покажет вспомогательную классификацию по ключевым словам:

```
Всего без адреса: 661. Подсказка по эвристике:
  • Жилые номера / Гостиница            112 шт.
  • Медицина / Лечение                   62 шт.
  • Ресторан / Кухня / Бар               34 шт.
  ...
```

Затем по каждой группе предложит создать бизнес-единицу:

```
  Группа: Жилые номера / Гостиница (112 объектов)
  Примеры: Торшер 01_1192 · Торшер 01_1193 · Кровать односпальная · ...
  Имя BU / '-' / '=bu_id' [Жилые номера / Гостиница]:
```

**Варианты ответа:**

| Ввод | Что произойдёт |
|---|---|
| **Enter** | Создать бизнес-единицу с предложенным именем (по умолчанию = название группы) |
| **`Корпус А`** (любое имя) | Создать BU с заданным именем |
| **`-`** | Пропустить группу: оборудование останется без BU (можно назначить позже) |
| **`=bu_korpus_gamma`** | Присоединить группу к уже созданной BU (используйте id из Фазы А, скрипт показывает их при создании) |

**Совет:** если не уверены — оставляйте имена по умолчанию. Скорректировать
названия можно позже, отредактировав JSON или повторно запустив скрипт.

## 6. Результат

В папке `_data/` болванки появится файл:

```
_data/structure_sanatoriy_sosnovaya_roscha_20260516_073621.json
```

В консоли — итоговое саммари:

```
══════════════════════════════════════════════════════════════════════
ГОТОВО
══════════════════════════════════════════════════════════════════════
  Файл: D:\Проекты\..._data\structure_sanatoriy_..._20260516_073621.json
  Записей: ОС=711 · BU=11 · кадастр=32 · аренда=8
```

**Что внутри (если открыть в текстовом редакторе):**

- `enterprise` — данные организации.
- `business_units` — бизнес-единицы, которые вы только что создали.
- `cadastre_objects` — кадастровые объекты (заглушки или с данными ЕГРН).
- `equipment` — все 711 ОС с привязкой к BU и кадастровым объектам.
- `lease_contracts` — договоры аренды (по строкам счёта `01.К`) с суммами
  начисленных арендных платежей за период.

## 7. Повторный запуск (обогащение)

Через несколько дней / после правок в 1С:

1. Сформировать **новый** `.xlsx` с актуальной ОСВ и положить в `ОСВ/`.
2. Запустить скрипт снова, в Вопросе 2 указать **предыдущий**
   `structure_*.json` — все ваши подтверждённые названия бизнес-единиц
   сохранятся. Новые ОС просто добавятся.
3. (Опционально) указать обновлённую папку с выписками ЕГРН — кадастровые
   объекты получат полные адреса и этажность.

## 8. Частые проблемы

### «Файл не найден или не `.xlsx`»

Проверьте путь и расширение. В Проводнике включите отображение
расширений (Вид → ☑ Расширения имён файлов).

### «`ModuleNotFoundError: No module named 'openpyxl'`»

Виртуальное окружение не активно или библиотека не установлена:

```powershell
pip install openpyxl
```

Если в начале строки терминала **нет** `(venv)` — выберите интерпретатор
заново через `Ctrl+Shift+P → Python: Select Interpreter`.

### «`01.К` не распознан / арендные платежи отсутствуют»

Убедитесь, что в файле ОСВ символ — именно **кириллическая К** (как
в «Касса»), а не латинская K. В 1С это всегда кириллица — обычно
проблема в том, что счёт был вручную переименован.

### Кодировка консоли — кракозябры

В VS Code: `Ctrl+Shift+P → Terminal: Select Default Profile → PowerShell`.
Если запускается `cmd.exe`, выполните перед запуском скрипта:

```cmd
chcp 65001
```

### Скрипт «висит» после вопросов

После третьего вопроса начинается интерактивный диалог по бизнес-единицам.
Проверьте, что курсор стоит в нижней строке терминала — нажмите Enter.

### «Не нашёл папку Не_распределено/»

Если запускали меню 3 в `pirushin_sosn_rocha_07_init_project_v1.py` —
эта папка лежит **в корне болванки**, рядом с `ОСВ/`, `Фотографии/`,
`json/`, а не внутри `Фотографии/`. Если её нет — создайте вручную
или запустите меню 1 («Создание структуры»).
