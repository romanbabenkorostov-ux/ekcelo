# Текстовые шаблоны (по абзацам) и JSON структура для блока «Развёрнутое описание» для конкретных площадок, чтобы сразу заложить их в движок генерации описаний.

Ниже — JSON-структура для блока «Развёрнутое описание» и набор готовых абзацных шаблонов, адаптируемых под ГИС Торги, Росэлторг, Сбербанк‑АСТ и другие ЭТП (по сути, различаются только длиной и «официальностью» формулировок).[^1][^2][^3]

## Абзацные шаблоны: ГИС Торги (официальный, сжатый стиль)

### Абзац 1. Идентификация и основные параметры

Шаблон:

> Объектом торгов является {identity.title} {identity.purpose? "коммерческого/жилого/иного назначения"} общей площадью {identity.area_total_sqm} кв. м, расположенное на {location.floor? "этаже/этажах"} {building.floors_total? "-этажного"} {building.building_type} здания по адресу: {location.region}, {location.locality}, {location.street}, {location.house}{location.building? ", " + location.building}{location.room? ", " + location.room}. Кадастровый номер объекта: {identity.cadastral_number}.

Если участок:

> Объектом торгов является земельный участок площадью {identity.area_land_sqm} кв. м, категория земель — {legal.zoning / permitted}, расположенный по адресу: {…}. Кадастровый номер: {identity.cadastral_number}.

### Абзац 2. Местоположение и окружение

> Объект расположен в {location.environment_short} {location.locality}. Доступ к объекту обеспечен по {location.transport_access} улично‑дорожной сети; {location.landmark? "в непосредственной близости расположены " + location.landmark + "." : ""}

Например:

> Объект расположен в зоне смешанной жилой и деловой застройки центральной части города. Доступ к объекту обеспечен по улично‑дорожной сети районного значения; в 5 минутах пешком находится станция метро «Тверская».[^5][^6]

### Абзац 3. Сведения о здании и инженерной инфраструктуре

Объект подключен к основным инженерным сетям: электроснабжение — {building.engineering.electricity}, водоснабжение — {building.engineering.water}, водоотведение — {building.engineering.sewerage}, отопление — {building.engineering.heating}{building.engineering.gas? ", газоснабжение — " + building.engineering.gas : ""}{building.engineering.telecom? ", телекоммуникации — " + building.engineering.telecom : ""}.

Высота потолков составляет {layout_and_condition.ceiling_height_m} м. Отделка — {layout_and_condition.finish_level}, состояние отделки оценивается как {layout_and_condition.finish_state}. {layout_and_condition.entry_group? "Входная группа: " + layout_and_condition.entry_group + ". " : ""}{layout_and_condition.current_condition_comment? layout_and_condition.current_condition_comment : ""}

### Абзац 5. Правовой статус и использование

## Генерация `generated_text.short` и `generated_text.full`

Для движка генерации удобно разделить:

- `short` — 2–3 абзаца (абз. 1–2), выводятся прямо в карточке лота на ЭТП.
- `full` — 4–6 абзацев, выводятся во встраиваемом окне (iframe) твоей системы или в приложении‑PDF.

Пример логики:

1) Определяем платформу и режим:

- `platform = "torgi.gov.ru", platform_mode = "short"` → берём абз. 1–3 ГИС‑шаблона.
- `platform = "roseltorg.ru", platform_mode = "full"` → абз. 1–6 Росэлторг‑шаблона.
- `platform = "sberbank-ast.ru"` → набор Сбербанк‑АСТ‑шаблонов, дополненный фразой про банкротство/приватизацию при наличии.

2) Собираем текст:

- Заполняем плейсхолдеры из JSON.
- Пропускаем фрагменты, если данных нет (условные вставки в шаблоне).

3) Сохраняем результат:
```json
"generated_text": {
  "short": "…2–3 абзаца…",
  "full": "…4–6 абзацев…",
  "version": 1
}
```
