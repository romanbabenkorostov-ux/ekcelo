# EkceloFoto — Руководство по рефакторингу (47a)

> Цель крупного рефакторинга (2.10): из императивного одностраничника →
> **single source of truth (Store) + materialized views + MVC + observer/pubsub**,
> с сохранением 100% текущего UI/UX. Делается несколькими несвязными кодерами,
> поэтому: маленькие шаги, каждый шаг зелёные тесты (`dev/tests`), сверка с
> замороженной `v2961.html` по URL-баннеру.

## Целевая архитектура

- **Model — `state.js` (Store, SSOT)**: единый объект состояния:
  `photos[]`, `kmlLayers[]`, `cadastre{groups,active,opacity}`, `ui{tab,activeIdx}`,
  `editors{ce,ep}`. Только Store мутирует состояние; мутаторы валидируют и
  эмитят события.
- **Observer — `events.js`**: крошечный pub/sub (`on/off/emit`). События:
  `photo:added|removed|enriched`, `kml:added|toggled|removed`,
  `cadastre:changed`, `ui:tab`, `lightbox:open|close`.
- **View — рендер-функции** (чистые, идемпотентные «materialized views»):
  `renderPhotoMarkers()`, `renderMarksList()`, `renderStats()`, `renderExportMenu()`,
  `renderLightbox()`. Подписаны на события, перерисовывают свой кусок DOM/Leaflet
  из Store. Никаких прямых мутаций состояния из View.
- **Controller** — из инлайновых `onclick=` в именованные хэндлеры: читают ввод,
  вызывают мутатор Store, View сам перерисуется по событию.

## Порядок (инкрементально, без big-bang)

1. Ввести `events.js` и `Store` рядом со старым кодом; **зеркалить** запись:
   старые функции дополнительно пишут в Store и эмитят. Поведение не меняется.
2. Перевести по одному View на чтение из Store + подписку (начать с `renderStats`).
3. Убрать дублирующую старую запись, когда View переехал.
4. Вынести `onclick=` в Controller-функции (по меню за раз).
5. Удалять мёртвый старый путь только когда тесты зелёные и сверка с `v2961.html` ок.

## Высокоценные мишени (с якорями)

- **Баг lifecycle фото+KML** (известное ограничение, см. `TASKSPEC_45_46_47.md`):
  `toggleKMLLayer` (~5369) снимает только `layerGroup`, фото-маркеры из `photos[]`
  живут отдельно → слой «не выключается». В новой модели слой владеет всеми
  своими сущностями (фото+placemark) в одной группе.
- **`_handleMapAreaClick`** (~2024–2497, ~470 строк): разнести на
  `handlePhotoClick/handleKMLClick/handleMapClick`.
- **`exportData`** роутер (~9141) + эмиттеры: вынести в `ExportManager`.
- **Лайтбокс** (~6178–6511): данные/представление перемешаны, развести.
- **`_dedupParsedPlacemarks`** (~5068–5158): стратегии дедупа (photo-pin / cadnum)
  разнести по интерфейсу.

## Жёсткие запреты

- **CSS шва `index.html` ~457–468** (защита 400–456) — не трогать; любое
  изменение рядом → полный чеклист регрессии (`CHECKLIST.md`).
- **`sw.js CACHE_NAME`** не менять (вайп кэша).
- **Код невыключения слоёв** — оставлен как известное ограничение; в рефакторинге
  чинится архитектурно (владение группой), не точечным патчем в 2.9.x.
- Стиль дома компактный (где-то `==` намеренно) — не переписывать стиль массово,
  линт advisory.
