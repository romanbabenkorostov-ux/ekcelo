## 45 — Регрессия шва + невыключение слоёв

- Белые полосы между плитками вернулись в 2.9.62 (рефактор `_objectIndex`, BUG 44).
  Пользователь откатил на 2.9.61. **Решение: работаем от 2.9.61, CSS-блок шва
  `index.html` ~457–468 НЕ трогаем** (см. `CHECKLIST.md` A).
- **Известное НЕустранённое ограничение:** слои не выключаются полностью —
  фигуры, загруженные из нескольких дублирующих KMZ, остаются (нарушение
  идемпотентности; `toggleKMLLayer` снимает только `layerGroup`, фото-маркеры
  отдельно). **РЕШЕНИЕ: оставлено как есть.** Workaround пользователя — грузить
  только один KMZ. Чинится архитектурно при рефакторинге 47a (владение группой),
  не точечным патчем в 2.9.x.

## 46a — Хранилище тайлов ✓ сделано в 2.9.62

## 46b — Префетч z17 центральных 70% ✓ сделано в 2.9.62

При включённом кадастре, независимо от выбранного zoom, догружаем z17-тайлы для
центральных 70% вьюпорта (центр ±35%) в SW-кэш. Идемпотентно (повтор — из кэша),
троттлинг (≤120/проход, конкуренция 6, debounce 400 мс, отмена устаревших
проходов по токену), хук на `moveend/zoomend` и при включении кадастра. URL —
**единый `_nspdWmsUrl`** (тот же, что в `createTile`), иначе ключ кэша SW не
совпадёт. Проверено ментально: Referer через `fetch{referrer}` работает в
Chrome/Firefox/Яндекс (Chromium); тайлы напрямую, не через прокси.

## 46c — Кнопки обмена тайлами ✓ сделано в 2.9.62

«Экспорт данных» → «Выгрузить тайлы Росреестра z17»: ZIP `tiles/NNNNN.png` +
`manifest.json` (`[{file,url}]`, url = точный ключ кэша). «Загрузить
фото/объекты» → «Загрузить чужие тайлы z17»: распаковка + идемпотентный импорт в
SW-кэш (обогащение — недостающее добавляем, существующее не трогаем). Протокол
SW: `dumpZ17Tiles`/`importZ17Tiles` (чанки ≤200, transferable ArrayBuffer).
