# SPEC — Роли (pro / view / embed), мягкое гейтирование, встраивание в iframe

> Статус: СПЕЦИФИКАЦИЯ (реализация отложена по решению владельца).
> Не меняет `index.html`. Готовится к рефакторингу 47a. Версия-цель: 2.10.
> Среда без backend (GitHub Pages, весь код — в браузере).

## 1. Контекст и цель

Нужны две роли:
- **pro** (текущее поведение) — просмотр + импорт + редактирование + экспорт.
- **view** (усечённая) — только просмотр + импорт. Без правки и экспорта.
- **embed** — `view` без «обвязки» (шапка/меню), пригодная для `<iframe>` на
  стороннем сайте.

## 2. Граница безопасности (зафиксировать явно)

Это **мягкое гейтирование UI**, НЕ криптозащита. Весь код уходит в браузер;
проверка пароля обходится через DevTools. Подходит, потому что данные не
секретны, задача — чистый просмотрщик для обычных пользователей и защита от
случайной правки, а не enforcement. Жёсткое разделение (физически нет кода
правки/экспорта) и реальная авторизация возможны только: (а) отдельной
viewer-сборкой из рефакторинга 47a, либо (б) backend (см. §8 React+FastAPI).

## 3. Резолвинг роли (приоритет сверху вниз)

1. `?mode=embed` в URL → роль `embed` (жёстко; localStorage игнорируется).
2. `?mode=view` в URL → роль `view`.
3. `?mode=pro` в URL → роль `pro` (для удобной про-ссылки).
4. `localStorage['ekcelo_role'] === 'pro'` (выставлен после ввода пароля) → `pro`.
5. Иначе — `DEFAULT_ROLE` (константа конфигурации).

`DEFAULT_ROLE` — рекомендуется `'pro'`, чтобы НЕ ломать текущих пользователей
основного сайта; публичный/встраиваемый доступ идёт явными ссылками
`?mode=view` / `?mode=embed`. (Альтернатива — `'view'` с про-разблокировкой;
выбор владельца, задокументировать в конфиге одной строкой.)

## 4. Мягкая разблокировка pro (пароль)

- Конфиг-блок (рядом с версией, верх `<script>`):
  ```
  const ROLE_CFG = {
    DEFAULT_ROLE: 'pro',                 // 'pro' | 'view'
    PRO_PASS_SHA256: '<hex sha256>',     // хэш парольной фразы (обфускация)
    EMBED_ALLOW_PARENT: ['*']            // для будущего postMessage allowlist
  };
  ```
- UI: пункт «🔑 Режим профи…» в существующем меню (или иконка-замок рядом с
  `#ver-switch`). Ввод фразы → `crypto.subtle.digest('SHA-256', …)` →
  сравнить с `PRO_PASS_SHA256` → при совпадении
  `localStorage['ekcelo_role']='pro'`, перезагрузить применение роли (без
  перезагрузки страницы — см. §6 «динамическое применение»). «Выйти из про» →
  очистить ключ → роль `view`.
- Честная подпись в диалоге: «Это переключатель интерфейса, не защита данных».

## 5. Что именно отключает роль (точный список)

«view» и «embed» = НЕ только спрятать CSS-классом, но и **обезвредить
обработчики** (ранний `return` в роль-гейте), чтобы нельзя было дёрнуть из
консоли случайно/через старый UI.

**Скрыть + обезвредить в view/embed (правка и экспорт):**
- Экспорт: кнопка `#export-btn` / `#export-wrap`, `toggleExportMenu`,
  `exportData`, `_exportCadastreTiles` (46c «Выгрузить тайлы…»).
- Редактирование контуров/точек: кнопки-карандаши в инфо-карточке
  (`editBtn` в рендере карточки), `_editContourFromCard`,
  `_editExpCoordFromCard`, `startContourEditor`, `startExplanationPoint`,
  тулбары контура/пояснения, `toggleContourShiftMode`, `_commitEdited
  GeometryInPlace`, `deleteContourShape`, `moveContourToOtherProject`.
- Создание объектов с карты: «💬 Добавить пояснение», «📍 Запросить кадастр»
  (если оно создаёт/сохраняет), действия инфо-карточки, меняющие данные.
- EXIF/координаты фото: ручной GPS, `writeEXIFWithGPS`, перетаскивание
  temp-маркера, авто-EXIF чекбокс.
- KMZ-тайлы экспорт (46c). Очистка кэша — оставить (не правка данных).

**Оставить в view (просмотр + импорт):**
- Загрузка фото/объектов: всё меню «Загрузить фото/объекты» (локально,
  папка, Я.Диск, GDrive, KML, KMZ), drag-n-drop, «Загрузить чужие тайлы z17».
- Слои/кадастр/НСПД, поиск по кадномеру, таймлайн, лайтбокс, темы,
  переключатель версий, прозрачность, линейка, дерево папок (только просмотр).

**Точка реализации:** один хелпер `roleAllows(cap)` где `cap ∈
{'edit','export'}`; каждый обработчик правки/экспорта начинается с
`if(!roleAllows('edit')) return;`. Рендер UI: класс на `<html>`
(`role-view` / `role-embed`) + CSS прячет кнопки; но логика-гейт —
обязательна (CSS недостаточно).

## 6. Динамическое применение роли

`applyRole(role)` ставит класс на `<html>`, прячет/показывает узлы, и НЕ
требует reload (роль читается гейтом в рантайме). Вызвать при старте и при
смене (вход/выход про). Идемпотентно.

## 7. Режим embed (iframe)

- `?mode=embed` ⇒ роль `embed` = `view` + дополнительно:
  - Скрыть `#header` (или оставить только лого+слои — настраиваемо
    `ROLE_CFG.EMBED_CHROME='none'|'min'`), скрыть `#ver-switch`.
  - Отключить FSA-пикеры, не работающие в кросс-доменном iframe
    (`showDirectoryPicker`/`showSaveFilePicker`) — код уже имеет fallback
    на `<input>`; в embed принудительно fallback, без try-catch-сюрпризов.
  - Карта/поиск/кадастр/лайтбокс работают.
- Сниппет для стороннего сайта:
  ```html
  <iframe src="https://<site>/index.html?mode=embed"
          style="width:100%;height:600px;border:0" loading="lazy"
          allow="clipboard-write"></iframe>
  ```
- Кросс-домен: CDN (Leaflet/exifr/JSZip/XLSX) и Cloudflare-прокси уже
  кросс-доменные — работают в iframe. GitHub Pages по умолчанию НЕ ставит
  `X-Frame-Options` → фрейминг разрешён. Не добавлять frame-busting.
- Опционально (фаза 2): `postMessage`-API хосту — `{type:'ekcelo:ready'}`,
  приём `{type:'ekcelo:loadKML', url}` с allowlist `EMBED_ALLOW_PARENT`.
  Вынести в отдельный мини-спек при необходимости.

## 8. Будущий реинжиниринг на React + FastAPI — осуществимо?

**Да, осуществимо и логично.**
- **Frontend → React.** Текущая модель данных (`kmlLayers[]`, `photos[]`,
  canonical `kmlText`, KML↔GeoJSON) переносима. Это ровно цель рефакторинга
  47a (см. `dev/REFACTOR_GUIDE.md`): single source of truth + observer →
  напрямую ложится на React-store (Zustand/Redux) + компоненты + Leaflet
  через `react-leaflet` или императивный ref. Роли из этой спеки становятся
  пропсами/контекстом; `roleAllows` → хук `useRole()`.
- **Backend → FastAPI.** Даёт то, чего нельзя на чистом клиенте:
  - НАСТОЯЩАЯ авторизация (JWT/сессии), серверный enforcement ролей,
    эндпоинты только-чтение для `view`;
  - многопользовательность, постоянное хранилище (Postgres/PostGIS),
    серверная дедупликация/идемпотентность из нескольких источников
    (закрывает известное ограничение задания 45);
  - серверный тайл-прокси/кэш Росреестра (заменяет Cloudflare worker + sw.js);
  - embed становится SDK с токеном вместо «мягкого» гейта.
- **Цена/последствия:** теряется свойство «без сервера, бесплатный хостинг
  GitHub Pages» — нужен хостинг API, БД, деплой, мониторинг, стоимость.
  Миграция поэтапная: 1) 47a-рефакторинг (чистая модель) → 2) обёртка в
  React, тот же клиентский режим → 3) ввод FastAPI как опционального
  backend (фичефлаг), клиент работает и без него → 4) перенос enforcement
  ролей/хранилища на сервер.
- **Совместимость спеки:** концепции `mode/role`, `embed`, `roleAllows`
  спроектированы так, чтобы при переходе стать серверными ролями и SDK без
  переписывания смысловой модели — только смена источника правды (клиент →
  API).

## 9. Конфиг, дефолты, edge-cases

- `?mode=embed` всегда побеждает сохранённый pro (встраивание на чужом сайте
  не должно «протекать» в pro-режим).
- Неизвестный `?mode=` → трактовать как `DEFAULT_ROLE`.
- Печать/экспорт PDF браузером — вне зоны (не отключаемо, не данные).
- localStorage недоступен (приватный режим) → роль из URL/дефолта, без краша.
- Если включён 47b `v2961.html` — спека применяется ТОЛЬКО к актуальному
  `index.html`; замороженная страница не трогается.

## 11. План тестов (ручной, т.к. в песочнице нет браузера)

1. `index.html` без параметров → роль `DEFAULT_ROLE`; pro = всё как сейчас.
2. `?mode=view` → нет кнопок экспорта/правки; импорт фото/KML/KMZ работает;
   попытка вызвать `exportData()`/`_editContourFromCard()` из консоли →
   тихий no-op (гейт).
3. `?mode=embed` в `<iframe>` на тестовой странице → нет шапки/баннера,
   карта+поиск+кадастр работают, FSA-диалоги не всплывают (fallback).
4. Ввод пароля → роль `pro`, появились правка/экспорт; «Выйти» → снова `view`.
5. `?mode=embed` при сохранённом `ekcelo_role=pro` → всё равно `embed`.
6. Тёмная/светлая/системная темы, маркеры/контуры на месте; шов не вернулся.
