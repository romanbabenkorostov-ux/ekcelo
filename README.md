# EkceloFoto — EXIF геолокация и данные опыта

Одностраничный мощный визуализатор GPS-меток и агрохимических данных из EXIF фотографий.  
Работает полностью в браузере — без сервера, без установки.

🌐 **[Открыть сайт](https://romanbabenkorostov-ux.github.io/ekcelo/)**

---

## Возможности

### Загрузка фотографий
- Drag & drop или выбор JPG-файлов с компьютера
- Google Drive (папка) — множественная загрузка через API-ключ
- Прямые ссылки на файлы:
  - **Yandex Disk** (`disk.yandex.ru/i/...` и `yadi.sk/i/...`)
  - **Google Drive**
  - **S3 Timeweb Cloud** (`https://s3.twcstorage.ru/...`)
- **Deeplink** — открытие одного или нескольких фото по ссылке

### Карта и визуализация
- Автоматическое чтение GPS-координат из EXIF
- Конус угла съёмки на карте (GPSImgDirection)
- Автоподгонка масштаба под все точки
- Клик по маркеру → подробная карточка фото
- Полноэкранный просмотр с навигацией ← →
- Переключение слоёв: OSM / Спутник (Esri)
- Кадастровый слой НСПД (nspd.gov.ru) с регулировкой прозрачности
- Клик по карте → информация о кадастровых участках

### Ручной GPS
- Для фото без координат или без угла — кнопка **«Установить»**
- Два способа ввода координат:
  - Клик по карте
  - Текстовый ввод (десятичный или DMS-формат): `47.373533, 39.899542` или `47°22'24.72"С 39°53'58.35"В`
- Интерактивный слайдер угла съёмки (0°–360°) с живым обновлением маркера
- Поддержка колёсика мыши
- Запись координат + угла прямо в EXIF файла (piexifjs)
- Автоматическое скачивание модифицированного JPEG

### Данные агрохимического опыта (GPP UserComment)
- Полная поддержка обогащённого тега **UserComment** (GPP формат, версия 1+)
- Карточка фото содержит **4 структурированные секции**:

#### 📍 Секция «Геолокация и съёмка»
- Координаты (с кнопкой «Копировать»)
- Угол съёмки (с кнопкой «Установить», если отсутствует)
- Высота над уровнем моря (если есть в EXIF)
- Дата съёмки

#### 📋 Секция «Данные опыта»
- Номер опыта, организация, оператор
- Культура, сорт / гибрид, поколение, BBCH
- **Дата сева** (`sow_date`)
- **Дата всходов** (`emerge_date`)

#### 💧 Секция «Влажность почвы»
- Влага 0–20 см, 0–60 см, 0–100 см
- **Влага 0–200 см** (`m200`) — новое поле

#### 🌡 Секция «Суммы температур»
- **ГДД ≥5°C** (`gdd5`)
- **ГДД ≥10°C** (`gdd10`)
- **ГДД ≥15°C** (`gdd15`)

В нижней части карточки:
- Кнопка **«📋 Копировать JSON»** — копирует полный GPP объект в буфер
- Кнопка **«🌐 Открыть опыт»** — ссылка на основной сайт из `invite_url` (если есть)

### Кнопка «Зайти на основной сайт»
- Появляется автоматически в **шапке приложения** при загрузке любого фото с полем `invite_url` в GPP UserComment
- Открывает основной сайт ГеоФотоПоле в новой вкладке
- Исчезает при удалении всех фото или загрузке фото без invite_url

### Поделиться и deeplink
- Кнопка **🔗 Поделиться этими фото**
- Поддержка нескольких фото в одной ссылке
- Форматы:
  - `?photo=https://...`
  - `?photo1=URL1&photo2=URL2&name1=Имя1`
  - `?photolist=URL_файла_JSON`
- Ограничение длины ссылки ≈ 2000 символов
- Автоматическое открытие первого фото при переходе по deeplink

## GPP UserComment — поддерживаемый формат

```json
{
  "app": "gpp",
  "v": 1,
  "exp": {
    "doc":         "ОП-2024-001",
    "org":         "ООО АгроТех",
    "op":          "petrov_iv",
    "dt":          "2024-05-15",
    "crop":        "Пшеница озимая",
    "variety":     "Танаис",
    "gen":         "ОС",
    "bbch":        "30",
    "seeded":      true,
    "sow_date":    "2023-10-12",
    "emerge_date": "2023-10-20",
    "m20":  48,
    "m60":  112,
    "m100": 195,
    "m200": 480,
    "gdd5":  312.4,
    "gdd10": 201.8,
    "gdd15": 98.2,
    "invite_url": "https://geofield.example.com/experiments/42"
  },
  "photo": {
    "gnss_lat": 47.3735,
    "gnss_lon": 39.8995,
    "bearing":  142.5
  }
}
```

Все поля кроме `app`, `v`, `exp.doc` — опциональны. Отсутствующие поля не отображаются.

## Примеры deeplink-ссылок

**Один файл:**
```
?photo=https://disk.yandex.ru/i/vogM25OyU9waFA
?photo=https://drive.google.com/file/d/1cEqjm.../view
?photo=https://s3.twcstorage.ru/bucket/photo.jpg
```

**Несколько файлов:**
```
?photo1=https://disk.yandex.ru/i/vogM25OyU9waFA&name1=Фото1
&photo2=https://disk.yandex.ru/i/EPMF7OBQgcVuvQ&name2=Фото2
```

**Список из файла:**
```
?photolist=https://disk.yandex.ru/i/XXXXXX  (файл .json со списком URL)
```

## Развёртывание на GitHub Pages

1. Форкните репозиторий или создайте новый
2. Загрузите `index.html`, `README.md`, `worker.js`, `_config.yml` в корень
3. В настройках репозитория → **Pages** → Source: `main` branch / `/ (root)`
4. Сайт доступен по адресу `https://{ваш-логин}.github.io/{репозиторий}/`

### Настройка CORS-прокси для Яндекс.Диска

1. Перейдите на [Cloudflare Workers](https://dash.cloudflare.com) → Create Worker
2. Назовите воркер `ekcelo-proxy`, нажмите Deploy
3. Нажмите Edit code, замените содержимое на `worker.js` из репозитория
4. Скопируйте URL воркера вида `https://ekcelo-proxy.YOUR.workers.dev`
5. Вставьте в `index.html` в константу `YANDEX_PROXY` (первые строки `<script>`)
